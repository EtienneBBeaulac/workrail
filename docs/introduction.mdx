---
title: "What is WorkRail?"
description: "MCP workflow enforcement server -- guides AI agents step-by-step through structured tasks"
---

WorkRail is an MCP server that enforces step-by-step workflows for AI agents. Instead of letting Claude or other agents race to a solution, WorkRail drip-feeds instructions one step at a time through the [Model Context Protocol](https://modelcontextprotocol.org). The agent completes each step before seeing the next -- it cannot skip ahead because it does not know what comes next.

## Quick Start

Add WorkRail to your MCP client configuration:

```json
{
  "mcpServers": {
    "workrail": {
      "command": "npx",
      "args": ["-y", "@exaudeus/workrail"]
    }
  }
}
```

Then prompt your agent:

```
Use the bug-investigation workflow to debug this auth issue
```

WorkRail discovers available workflows automatically, starts at step 1, and walks the agent through systematically.

## How it works

```
Agent                     WorkRail
  │                          │
  │  start_workflow()        │
  │─────────────────────────>│
  │                          │
  │   Step 1: Understand     │
  │      the problem         │
  │<─────────────────────────│
  │                          │
  │  continue_workflow()     │
  │─────────────────────────>│
  │                          │
  │   Step 2: Plan your      │
  │      investigation       │
  │<─────────────────────────│
```

Sessions are durable -- work is saved to disk at every step and can be resumed across conversations with `resume_session`.

## What's included

WorkRail ships with 30+ workflows for coding, review, debugging, documentation, and more. See the [Workflows](/docs/workflows) page for the full list.

## Write your own

Drop a JSON file in `~/.workrail/workflows/` and WorkRail discovers it automatically:

```json
{
  "id": "my-review-checklist",
  "name": "Team Code Review",
  "version": "1.0.0",
  "steps": [
    {
      "id": "check-tests",
      "title": "Verify Test Coverage",
      "prompt": "List all new code paths that lack test coverage."
    }
  ]
}
```

See the [Authoring Guide](/docs/authoring-v2) for the full schema and workflow design principles.
