---
title: "Session Report"
description: "Export session history and metrics as machine-readable JSON for analysis, dashboards, and cost tracking"
---

`workrail report` reads your local session store and outputs a JSON summary to stdout. It covers session outcomes, token usage, lines changed, steps completed, and per-workflow breakdowns -- everything you need to measure what your agent actually did.

No daemon required. No network calls. The report reads the same files the console dashboard uses.

## Basic usage

```sh
# Sessions from the last 30 days (default)
workrail report

# Last 7 days
workrail report --days 7

# Specific date range
workrail report --since 2026-05-01 --until 2026-05-31

# Write to a file instead of stdout
workrail report --out report.json
```

## Piping with jq

stdout is always clean JSON (warnings and progress go to stderr), so you can pipe directly:

```sh
# Total tokens used this month
workrail report | jq '.summary.totalInputTokens + .summary.totalOutputTokens'

# Sessions by outcome
workrail report | jq '.summary.outcomeBreakdown'

# List completed sessions with their goals
workrail report | jq '[.sessions[] | select(.metrics.status == "complete") | {goal, workflowId, date}]'

# Which workflows ran most often
workrail report | jq '.summary.workflowBreakdown | to_entries | sort_by(-.value)'

# Lines changed by language this month
workrail report | jq '.summary.languageBreakdown'
```

## Output structure

The report is a single JSON object:

```json
{
  "version": 1,
  "generatedAt": "2026-05-30T12:00:00.000Z",
  "dateRange": { "since": "2026-05-01", "until": "2026-05-30" },
  "sessions": [...],
  "summary": {...}
}
```

### `summary` fields

| Field | What it counts |
|---|---|
| `totalSessions` | Sessions in the date range |
| `completedSessions` | Sessions with status `complete` or `complete_with_gaps` |
| `totalDurationMs` | Combined wall-clock time across sessions |
| `totalInputTokens` | Tokens sent to the model |
| `totalOutputTokens` | Tokens generated by the model |
| `totalCacheReadTokens` | Tokens served from prompt cache |
| `totalCacheWriteTokens` | Tokens written to prompt cache |
| `totalLinesAdded` | Lines added across all committed diffs |
| `totalLinesRemoved` | Lines removed across all committed diffs |
| `totalFilesChanged` | Files touched across all committed diffs |
| `totalStepsCompleted` | Workflow steps completed |
| `totalRetriesCount` | Step retries across all sessions |
| `outcomeBreakdown` | Session count per outcome (`success`, `partial`, `abandoned`, `error`, `unknown`) |
| `workflowBreakdown` | Session count per workflow ID |
| `languageBreakdown` | Lines changed per file extension |

### `sessions` fields

Each session entry contains:

| Field | Description |
|---|---|
| `sessionId` | Unique session identifier |
| `workflowId` | The workflow that ran (`null` for legacy sessions) |
| `goal` | Human-readable goal for the session |
| `date` | Date of last activity (`YYYY-MM-DD`) |
| `repoRoot` | Absolute path to the workspace the session ran against |
| `triggerSource` | `"mcp"` for interactive sessions |
| `metrics` | Full per-session metrics (tokens, lines, steps, outcome, duration) |

## Options

| Flag | Default | Description |
|---|---|---|
| `--days <n>` | `30` | Sessions modified in the last N days |
| `--since <date>` | (computed from `--days`) | Override start date (`YYYY-MM-DD`); takes priority over `--days` |
| `--until <date>` | today | Override end date (`YYYY-MM-DD`, inclusive) |
| `--out <file>` | stdout | Write JSON to a file instead |

## Session limit

The report loads at most 500 sessions (most recently modified first). If your session store exceeds this, a warning is printed to stderr and the report covers the 500 most recent sessions. Use `--days` to narrow the window.

## Data location

Reports read from `~/.workrail/data/sessions/`. Set `WORKRAIL_DATA_DIR` to override:

```sh
WORKRAIL_DATA_DIR=/custom/path workrail report
```
